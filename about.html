Mission

왜 RE:Factory를 만들었는가

AI가 코드를 대신 작성하는 시대, 누구나 손쉽게 코드를 만들어내지만 그 안의 구조와 흐름을 온전히 이해하기는 점점 더 어려워지고 있습니다.

자동 생성된 코드는 종종 복잡하고 비효율적이어서, 문제가 발생했을 때 원인을 찾는 데 많은 시간이 듭니다. 비효율적인 코드는 결국 성능 저하와 운영 비용의 증가로 이어집니다.

RE:Factory는 이 문제를 정면으로 마주하기 위해 만들어졌습니다. 코드를 해부해 보여주고, 더 나은 형태로 다시 설계할 방향을 제시합니다.

What We Do

코드에서 인사이트로, 다섯 단계

RE:Factory는 코드 입력부터 개선 제안까지, 분석의 전 과정을 하나의 파이프라인으로 연결합니다.

  1. 01

    Code Input · 코드 입력

    ZIP 파일 업로드나 GitHub 저장소 연동으로 분석할 코드를 손쉽게 등록합니다.

  2. 02

    Static Analysis · 정적 분석

    tree-sitter 기반 파서로 Python, JavaScript, Java, C 등 다양한 언어를 동일한 방식으로 해석합니다.

  3. 03

    Structure Visualization · 구조 시각화

    함수 호출 관계와 모듈 의존 구조를 그래프로 시각화해 코드의 전체 흐름을 한눈에 보여줍니다.

  4. 04

    Quality Assessment · 품질 평가

    Cyclomatic Complexity, Code Smell, 중복 코드 등을 종합 분석해 코드 품질 점수를 산출합니다.

  5. 05

    Refactoring Suggestions · 리팩토링 제안

    탐지된 문제 영역에 대해 어떤 방향으로 개선할 수 있는지 구체적인 제안을 제시합니다.

Why RE:Factory

RE:Factory가 다른 이유

01

Language-Agnostic Analysis
언어를 가리지 않는 분석

tree-sitter 기반 범용 파서를 사용해 특정 언어에 종속되지 않습니다. Python, JavaScript, Java, C 등 다양한 언어의 프로젝트를 하나의 도구로 일관되게 진단합니다.

02

Two Views, One Codebase
두 가지 시야로 보는 같은 코드

개발자를 위한 '정밀 보기'와 비개발자를 위한 '쉬운 보기'를 함께 제공합니다. 역할과 배경이 다른 사람들도 같은 코드를 자신의 시선으로 이해할 수 있습니다.

03

Insight, Not Just Inspection
분석을 넘어 제안까지

문제를 발견하는 것에서 멈추지 않습니다. 탐지된 이슈마다 "이렇게 바꾸면 더 낫다"는 개선 방향을 함께 제시해 다음 행동으로 이어지도록 돕습니다.

Vision

누구나 코드를 이해할 수 있는 세상

RE:Factory는 코드가 더 이상 소수의 전유물이 아닌 시대를 지향합니다. 개발자든 아니든, 누구나 코드의 구조와 흐름을 이해하고 더 나은 방향을 그릴 수 있는 환경을 만듭니다.

모든 코드는 다시 더 나은 형태로 설계될 수 있다고 믿습니다.
